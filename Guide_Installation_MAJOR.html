▶ GUIDE D'INSTALLATION

Application Web — Club MAJOR · El Jadida (Mazagan)

Guide destiné aux non-développeurs · Avril 2026

Ce guide vous explique, pas à pas et sans jargon technique, comment installer et lancer l'application Club MAJOR sur votre ordinateur Windows.

⏰ Durée estimée : entre 30 et 45 minutes la première fois. Les fois suivantes, il faut moins de 30 secondes pour relancer l'application.

1 Installer Node.js 5 min

Node.js est le moteur qui permet à l'application de fonctionner sur votre ordinateur.

  1. Ouvrez votre navigateur (Chrome, Edge, Firefox) et allez sur nodejs.org
  2. Cliquez sur le gros bouton vert "LTS" (c'est la version stable recommandée)
  3. Lancez le fichier téléchargé et cliquez Suivant jusqu'à la fin sans rien modifier
  4. Redémarrez votre ordinateur après l'installation

Vérification

Ouvrez le menu Démarrer, tapez cmd, appuyez sur Entrée. Dans la fenêtre noire, tapez :

node --version
✓ Si vous voyez quelque chose comme v20.11.0, Node.js est bien installé.

2 Installer PostgreSQL (la base de données) 10 min

PostgreSQL est le logiciel qui stocke toutes les données du club (membres, paiements, événements...).

  1. Allez sur postgresql.org/download/windows
  2. Cliquez sur "Download the installer"
  3. Téléchargez la version la plus récente pour Windows (64-bit)
  4. Lancez l'installateur et suivez ces instructions :
    • Laissez tous les composants cochés par défaut
    • Dossier d'installation : laissez le chemin proposé
    • À l'étape Password : tapez un mot de passe — par exemple Major2025
    • ⚠ Notez ce mot de passe, vous en aurez besoin à l'étape 5 !
    • Port : laissez 5432
    • Locale : laissez la valeur par défaut
    • Cliquez Suivant puis Finish
  5. À la dernière fenêtre, décochez "Launch Stack Builder" et cliquez Terminer
✓ Pour vérifier : cherchez pgAdmin dans le menu Démarrer. Si l'application s'ouvre, PostgreSQL est installé.

3 Créer la base de données du club 3 min

  1. Ouvrez pgAdmin depuis le menu Démarrer
  2. Au premier lancement, il vous demande un "master password" — choisissez ce que vous voulez et notez-le
  3. Dans la colonne de gauche, déroulez : Servers → PostgreSQL → Databases
  4. Faites un clic droit sur Databases → Create → Database...
  5. Dans le champ Database, tapez exactement : clubmajor
  6. Cliquez Save
✓ La base de données clubmajor apparaît dans la liste à gauche.

4 Localiser le dossier du projet

Le projet se trouve dans le dossier e:\Site_Major sur votre ordinateur.

ℹ Si vous souhaitez déplacer ce dossier, faites-le maintenant avant de continuer.

5 Créer le fichier de configuration (.env) 5 min

Ce fichier indique à l'application comment se connecter à votre base de données.

  1. Ouvrez le dossier Site_Major dans l'Explorateur de fichiers
  2. Vous verrez un fichier nommé .env.example
  3. Faites un clic droit dessus → Copier
  4. Faites clic droit dans un espace vide → Coller
  5. Renommez la copie en .env (supprimez le .example)
⚠ Si Windows refuse un nom commençant par un point : ouvrez le Bloc-notes, faites Fichier → Enregistrer sous, naviguez jusqu'au dossier Site_Major, tapez .env dans "Nom de fichier", choisissez Tous les fichiers (*.*) dans "Type", puis Enregistrer.
  1. Ouvrez le fichier .env avec le Bloc-notes
  2. Trouvez la ligne DATABASE_URL et remplacez VOTRE_MOT_DE_PASSE par le mot de passe choisi à l'étape 2 :
DATABASE_URL="postgresql://postgres:Major2025@localhost:5432/clubmajor" NEXTAUTH_SECRET="MajorClub2025SecretKeyElJadidaMaroc" NEXTAUTH_URL="http://localhost:3000"
ℹ Remplacez Major2025 par le mot de passe exact que vous avez choisi lors de l'installation de PostgreSQL.
Les autres lignes (NEXTAUTH_SECRET, NEXTAUTH_URL) peuvent rester telles quelles.
  1. Sauvegardez le fichier (Ctrl+S) et fermez le Bloc-notes

6 Ouvrir le terminal dans le bon dossier

  1. Ouvrez l'Explorateur de fichiers
  2. Naviguez jusqu'au dossier Site_Major
  3. Cliquez dans la barre d'adresse en haut
  4. Tapez cmd et appuyez sur Entrée
✓ Une fenêtre noire s'ouvre avec e:\Site_Major> au début. C'est parfait.

7 Installer les dépendances du projet 5 min

Dans le terminal, tapez la commande suivante et appuyez sur Entrée :

npm install --legacy-peer-deps
ℹ Beaucoup de texte va défiler pendant 2 à 5 minutes. C'est normal — l'application télécharge les bibliothèques nécessaires. Attendez que le curseur réapparaisse.
✓ Quand c'est terminé, vous reverrez e:\Site_Major> et un message du type "added 500 packages".

8 Créer les tables de la base de données 2 min

Cette commande applique toute la structure de la base de données :

npx prisma migrate deploy
✓ Vous verrez "All migrations have been successfully applied". Les tables sont créées.
ℹ Si vous voyez une erreur de connexion, vérifiez que le mot de passe dans le fichier .env est bien identique à celui choisi lors de l'installation de PostgreSQL (étape 2).

9 Remplir avec les données de démonstration 1 min

Cette commande crée les comptes de démonstration, les membres, événements, articles de blog, etc. :

npx prisma db seed
✓ Vous verrez des messages de confirmation : "✅ Membres créés", "✅ Événements créés"...
⚠ Si vous voyez l'erreur command not found: tsx, tapez d'abord npm install tsx --save-dev puis relancez npx prisma db seed.

10 Lancer l'application ! ▶

npm run dev

Après quelques secondes, vous verrez :

▶ Next.js 14 - Local: http://localhost:3000 - Ready in 2.1s
🎉 L'application est lancée ! Ouvrez votre navigateur et allez sur :

    http://localhost:3000

🔁 Relancer l'application les prochaines fois

Vous n'avez plus besoin de refaire toutes ces étapes. À partir de maintenant :

  1. Ouvrir l'Explorateur de fichiers → aller dans le dossier Site_Major
  2. Cliquer dans la barre d'adresse → taper cmd → Entrée
  3. Taper npm run dev → Entrée
  4. Ouvrir http://localhost:3000 dans le navigateur
⚠ La fenêtre noire (terminal) doit rester ouverte tant que vous utilisez l'application. Si vous la fermez, l'application s'arrête.

⚙ Fonctionnalités optionnelles

Emails automatiques (notifications)

Pour activer l'envoi d'emails (confirmation d'inscription, validation de compte, rappels de paiement) :

  1. Créez un Mot de passe d'application Gmail : allez sur myaccount.google.com → Sécurité → Validation en 2 étapes → Mots de passe des applications → Créer "Club MAJOR"
  2. Ouvrez le fichier .env et remplissez les lignes SMTP :
SMTP_USER="votre.adresse@gmail.com" SMTP_PASS="xxxx xxxx xxxx xxxx" EMAIL_FROM="Club MAJOR <votre.adresse@gmail.com>"
ℹ Tant que SMTP n'est pas configuré, les emails sont simplement ignorés — le site fonctionne normalement.

Intégration Strava

Pour afficher les activités du club Strava sur la page /strava, renseignez dans .env :

STRAVA_CLIENT_ID="votre_id" STRAVA_CLIENT_SECRET="votre_secret" STRAVA_REFRESH_TOKEN="votre_token"
ℹ Ces informations se trouvent sur strava.com/settings/api après création d'une application Strava.

🔧 Problèmes fréquents et solutions

Message d'erreurCause probableSolution
'node' n'est pas reconnu Node.js pas installé correctement Réinstaller Node.js (étape 1) et redémarrer le PC
password authentication failed Mauvais mot de passe PostgreSQL dans .env Vérifier le mot de passe dans .env (ligne DATABASE_URL)
database "clubmajor" does not exist La base de données n'a pas été créée Refaire l'étape 3 dans pgAdmin
All migrations already applied Normal si déjà installé Pas d'action nécessaire, continuez à l'étape 9
Port 3000 is already in use Une autre application utilise le port 3000 Taper : npm run dev -- --port 3001 puis aller sur http://localhost:3001
Cannot find module npm install pas fait ou incomplet Relancer npm install --legacy-peer-deps (étape 7)
EPERM: operation not permitted Prisma bloqué par un processus actif Arrêter npm run dev (Ctrl+C dans le terminal) puis relancer la commande
Page blanche ou erreur rouge npm run dev ne tourne pas Vérifier que le terminal est ouvert et relancer npm run dev
